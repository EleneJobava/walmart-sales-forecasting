# walmart-sales-forecasting



მოდელი 1: XGBoost-ის ექსპერიმენტები და ანალიზი 
Xgboost-ი შევარჩიეთ როგორც ერთ-ერთი ყველაზე მძლავრი საბაზისო მოდელი დროითი მწკრივების პროგნოზირებისთვის. ექსპერიმენტის მიზანი იყო მოდელის არქიტექტურის სრულყოფა, Feature Engineering-ი და ოვერფიტის რისკების კონტროლი.

1. მონაცემების cleaning-ი და Feature Engineering
მონაცემების მომზადების ეტაპზე განსაკუთრებული ყურადღება დაეთმო დროითი მწკრივების სპეციფიკას. მონაცემებში არსებული 1285 უარყოფითი მნიშვნელობის მქონე ჩანაწერი შენარჩუნდა, რადგან ისინი წარმოადგენდნენ ლეგიტიმურ return-ებს და არა ტექნიკურ შეცდომას. შევქმენით სპეციალური კლასი WalmartFeatureEngineer, რომელმაც დააგენერირა 78-მდე ახალი ცვლადი. მათ შორის შეიქმნა დროითი Lag ცვლადები 1-დან 52 კვირამდე, მცოცავი საშუალოები/მაქსიმუმები (Rolling windows) და დღესასწაულებამდე დარჩენილი კვირების რაოდენობა.

2. Overfitting და Underfitting დიაგნოსტიკა
იმისათვის, რომ თავიდან აგვეცილებინა მოდელის მიერ დატის მემორიზაცია, ჩავატარეთ დეტალური Learning Curve-ების დიაგნოსტიკა სამ განსხვავებულ კონფიგურაციაზე.

* Underfit
ამ შემთხვევაში ჩვენ მოდელს მივეცით ძალიან მცირე რესურსი (მხოლოდ 30 ხე) და ავუკრძალეთ რთული კავშირების დანახვა
(სიღრმე შევზღუდეთ 2-მდე). დამატებით, ჩავრთეთ ძალიან მკაცრი ჯარიმა (Regularization, reg_lambda=50). მოდელი გამოვიდა ზედმეტად "პრიმიტიული". Walmart-ის გაყიდვებს აქვს ურთულესი სეზონურობა. 2-დონიანი ხე უბრალოდ ვერ იტევს ამხელა ინფორმაციას, ამიტომ მან ვერც სატრენინგო მონაცემები ისწავლა და ვერც ახალ (ვალიდაციის) მონაცემებზე აჩვენა კარგი შედეგი (WMAE 1862.0).

* Overfit
აქ მოდელს მივეცით უზარმაზარი რესურსი (1500 ღრმა ხე) და მოვუხსენით ყველა ჯარიმა/შეზღუდვა (reg_lambda=0). მან სატრენინგო მონაცემებზე ცდომილება ზუსტად 0.0-ზე ჩამოიყვანა. მოდელმა კი არ "ისწავლა" ზოგადი კანონზომიერებები, არამედ დაიზეპირა სატრენინგო სეტში არსებული ყველა ხმაური (Noise) და ანომალია. როდესაც მას მივეცით მომავლის უცნობი მონაცემები (ვალიდაცია), დაზეპირებულმა წესებმა აღარ იმუშავა და შეცდომა გაიზარდა 1886.2-მდე

* ოქროს შუალედი (Tuned 400 ხე, სიღრმე 7)
ეს არის ბალანსი, სადაც მოდელი საკმარისად ჭკვიანია, რომ ისწავლოს ტრენდი, მაგრამ შეზღუდულია ისე, რომ არ დაიზეპიროს ხმაური.

3. Time-Based Cross-Validation
ჩვენ ვქმნით ისეთ მახასიათებლებს, როგორიცაა lag_1 (წინა კვირის გაყიდვები) და მცოცავი საშუალოები (rolling_mean_4). მონაცემები რომ შემთხვევითად აგვერია, მოდელი ტრენინგის დროს გამოიყენებდა მაგალითად, 2012 წლის მარტის მონაცემებს 2012 წლის თებერვლის დასაპროგნოზებლად. ეს არის კლასიკური მონაცემთა გაჟონვა, მოდელი იგებს მომავალს წარსულის პროგნოზირებისთვის, რაც რეალურ ცხოვრებაში შეუძლებელია. ამის გამო გამოვიყენეთ TimeSeriesSplit რომელიც უზრუნველყოფს, რომ მოდელი ყოველთვის სწავლობს წარსულზე (მაგ: 2010-2011) და პროგნოზს აკეთებს მკაცრად მომავალზე.

CV-ის დროს ჰიპერპარამეტრების ტუნინგმა (HPO) მოგვცა ქულა 2148.59, მაგრამ ფინალურმა Holdout ტესტმა გვიჩვენა ბევრად უკეთესი შედეგი - 1404.79. ამ სხვაობის მიზეზი ის არის რომ WMAE მეტრიკაში დღესასწაულებს (Thanksgiving, Christmas) აქვთ 5-ჯერ დიდი წონა. კროს-ვალიდაციის 5 ფოლდი მოიცავდა მთელი წლის ციკლებს, თავისი დიდი დღესასწაულებით, სადაც მოდელს ბუნებრივად უჭირს ზუსტი პროგნოზის გაკეთება და ცდომილება იზრდება. ჩვენი ფინალური სატესტო ფანჯარა (კვირები 130-დან 143-მდე) უწევს 2012 წლის აგვისტოდან ოქტომბრის ბოლომდე პერიოდს. ამ პერიოდში არ არის დიდი, მაღალ-წონიანი დღესასწაულები, შესაბამისად, მოდელის აბსოლუტური ცდომილება ლოგიკურად ბევრად დაბალია (1404.79).  

დაბკა მოცმეულია ოვერფიტის და ანდეფიტის მრუდების ვიზუალური ანალიზი. წარმოდგენილი გრაფიკი ნათლად ასახავს მოდელის სწავლების დინამიკას მონაცემების მოცულობის ზრდასთან ერთად (X-ღერძი წარმოადგენს სატრენინგო ჩანაწერების რაოდენობას, Y-ღერძი WMAE ცდომილებას).

ტრენინგისა (ლურჯი) და ვალიდაციის (ნარინჯისფერი) მრუდები ერთმანეთთან ძალიან ახლოსაა, მაგრამ ორივე საკმაოდ მაღალ ნიშნულზეა (1700-1900 დიაპაზონში). სატრენინგო მონაცემების რაოდენობის ზრდასთან ერთად, ხაზები ფაქტობრივად ჰორიზონტალურია და არ უმჯობესდება. მოდელი ზედმეტად მარტივია საიმისოდ, რომ მონაცემებში არსებული რთული სეზონური პატერნები აღიქვას.

შუა გრაფიკში ლურჯი ხაზი აბსოლუტურ ნულზეა დაშვებული. მოდელი ტრენინგზე საერთოდ არ ცდება. სამაგიეროდ, ნარინჯისფერი ხაზი (ვალიდაცია) უზარმაზარ ნიშნულზეა განთავსებული და მონაცემების დამატებასთან ერთად ოდნავ ზემოთაც კი მიიწევს.

ბოლო გრაფიკზე ტრენინგის ცდომილება რეალისტურ ნიშნულზეა (ნულის ნაცვლად ~700-თან), რაც ნიშნავს, რომ მოდელმა შეწყვიტა ხმაურის დაზეპირება. მთავარი კი ისაა, რომ ნარინჯისფერი ხაზი (ვალიდაცია) მონაცემების დამატებასთან ერთად სტაბილურად ქვემოთ მოდის. დისტანცია მრუდებს შორის თანდათან ვიწროვდება.
